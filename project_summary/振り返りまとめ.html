ナレッジシェアリングシステム（MVPベータ版）開発振り返り

※ 本システムはMVP（Minimum Viable Product）・ベータ版として開発中です。
主要機能は実装済みですが、今後さらなる機能追加・改善・本番運用に向けた拡張が予定されています。

1. 大まかな要件

  • ナレッジ（知識）をMarkdown形式で登録・編集・削除・検索できる
  • WYSIWYGエディタでMarkdown未経験者も直感的に入力可能
  • タグ・カテゴリによる分類と検索
  • AIチャットによるナレッジベースQ&A（RAG方式）
  • SSO（Google）によるログイン・ユーザ管理（管理者/一般）
  • 削除は2段階認証で誤操作防止
  • 日本語UI・「Yu Gothic UI」フォント・ポップなデザイン
  • 無料クラウド（Render）で社内限定公開
※ 現時点ではMVPのため、AIチャットやSSOなど一部機能は今後拡張予定です。

2. 使用ライブラリ・技術

  • バックエンド: Python, FastAPI, Uvicorn, CORS Middleware
  • フロントエンド: React (TypeScript), Toast UI Editor, marked, DOMPurify
  • デプロイ: Render (Web Service/Static Site), GitHub
  • その他: npm, venv, PowerShell, Git

4. 苦労した点

  • React/Toast UI Editor/TypeScript/markedのバージョン整合性
  • PowerShellの実行ポリシーや仮想環境の権限問題
  • RenderでのNode/Python混在によるビルド・デプロイエラー
  • Markdownの箇条書き・装飾の正確な表示
  • タグの複数選択UIや削除フローのUX
  • サンプルデータの粒度・分割・業務フロー化

5. アピールポイント

  • 日本語UI・ポップなデザイン：社内利用を意識した親しみやすさ
  • WYSIWYGエディタ：Markdown未経験者も使いやすい
  • AIチャット：RAG方式でナレッジを活用したQ&A
  • SSO・ユーザ管理：セキュリティと利便性の両立
  • 無料クラウド活用：コストを抑えたスピード開発
